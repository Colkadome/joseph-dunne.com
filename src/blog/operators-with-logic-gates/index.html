| Operators with Logic Gates

I had a dumb idea the other day. What if I re-created operations (+, -, *) with logic gates. I would take two input numbers, A and B, and route their bits to an output, C.

Addition - the normal way.

1-bit addition.

Two numbers can be added with logic gates by using the "sum" and the "carry out":

Falstad link

As all logic gates can be replicated with NAND gates, we can make a NAND-only version:

Falstad Link

2-bit addition.

We then use the "carry out" when adding the 2nd bit:

Falstad link

And the NAND-only version:

Falstad link

3-bit addition.

For any larger number, you can start chaining the circuit.

Falstad link

1-bit addition (optimized)

We saw above that 1-bit addition can be achieved with 2 gates (or 6 NAND gates). We can get down to 5 NAND gates though.

Falstad link

2-bit addition (optimized)

Similar to 2-bit addition above, the Carry-on from 1-bit addition can be extended. Doing this can get you down to 14 NAND gates.

3-bit addition (random)

So usually the number of gates is 2 + 5 gates (or 5 + 9 NAND gates) meaning 3-bit addition needs at least 12 gates (or 23 NAND gates). Can we beat this by randomly generating a circuit?
